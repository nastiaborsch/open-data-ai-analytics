## 6. Виконані модулі

### 6.1. Модуль завантаження даних (`data_load`)
Створено файл `src/data_load.py`, який:
- знаходить файл даних у папці `data/raw/`
- підтримує читання CSV/ZIP
- виводить розмір таблиці
- виводить назви колонок
- виводить перші рядки даних
- зберігає preview у `reports/data_preview.csv`

### 6.2. Модуль перевірки якості даних (`data_quality_analysis`)
Створено файл `src/data_quality_analysis.py`, який:
- завантажує обмежену вибірку даних
- перевіряє типи даних
- аналізує пропуски
- підраховує дублікати
- формує описову статистику
- виконує додаткові перевірки полів (`value`, `dateTime`, `locationId`, `indicatorId`)
- зберігає текстовий звіт у `reports/data_quality_report.txt`

### 6.3. Модуль дослідження даних (`data_research`)
Створено файл `src/data_research.py`, який:
- виконує базовий дослідницький аналіз даних
- аналізує частоти `indicatorId`
- визначає топ-локації за кількістю записів
- виконує денну агрегацію значень `value`
- знаходить топ-20 найбільших значень `value` (потенційні аномалії)
- зберігає результати у файли `reports/*.csv` та `reports/data_research_summary.txt`

### 6.4. Модуль візуалізації (`visualization`)
Створено файл `src/visualization.py`, який:
- будує гістограму значень `value`
- будує boxplot для аналізу аномалій
- будує bar chart для топ-10 `locationId`
- зберігає графіки у `reports/figures/`

Було створено і використано такі feature-гілки:
